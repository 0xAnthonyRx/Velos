#  Velos: Agentic Governance & Shift-Left Security
**AWS 10,000 AIdeas Competition | Track: Workplace Efficiency | Team: 0xlabs**

Velos is a DevSecOps interceptor designed to close the AI-assisted security gap. It watches the semantic intent of generative AI coding agents and autonomously rewrites their guardrails in real-time, preventing misconfigurations (like Public S3 buckets) before a single line of infrastructure code is generated.

##  The Vision: Proactive Agentic Steering
Standard AI coding agents prioritize speed over enterprise security. Velos implements a "Shift-Left" governance model. Instead of scanning for vulnerabilities *after* code is written, Velos intercepts the architectural design phase, detects critical risks (OWASP A01), and physically injects hard security requirements into the agent's steering logic.



##  Architecture & AWS Stack

Velos relies on a three-tier "Chain of Trust" architecture, powered entirely within the AWS ecosystem:

| Component | AWS Service | Role in Velos |
| :--- | :--- | :--- |
| **Generative Agent** | **Kiro** | Translates the Markdown designs into deployable Terraform (`.tf`) infrastructure. |
| **Semantic Brain** | **Amazon Bedrock (Nova)** | Audits the architectural intent for OWASP Top 10 vulnerabilities (simulated locally for rapid prototyping). |
| **Final Auditor** | **Amazon Q** | Performs the final SAST scan to verify the Kiro-generated infrastructure achieves "Zero Taint" compliance. |

### The Workflow
1. **Semantic Interception:** Velos monitors `design.md`. If a vulnerable architecture (e.g., Public S3) is proposed, it flags the intent.
2. **Agentic Steering:** Velos autonomously injects unbreakable security rules into `steering.md` (e.g., *"CRITICAL: Use CloudFront OAC for any public file delivery"*).
3. **Autonomous Remediation:** Kiro reads the new rules, corrects its trajectory, and generates a fully secure Terraform implementation.

##  Technical Innovation & Optimization

### 1. Cost-Resilient Execution ($0.00 Footprint)
Scanning every file save via an LLM API introduces massive token waste and cost overruns. Velos solves this by implementing a highly optimized **Cryptographic Content Hashing** system (`hashlib.sha256`). Velos hashes the design file locally and only triggers the Bedrock semantic security engine if the content string fundamentally changes. This guarantees the tool runs seamlessly within the AWS Free Tier.

### 2. "Zero Taint" SAST Compliance
Velos itself is hardened to pass enterprise Static Application Security Testing (SAST). Custom path sanitization logic was written to eliminate dynamic pathing vulnerabilities, ensuring the Python orchestrator meets the strict "Zero Taint" requirements validated by Amazon Q.

##  Local Installation

Ensure you have Python 3.10+ and your AWS credentials configured.

```bash
# Clone the repository
git clone git@github.com:0xAnthonyRx/velos.git
cd velos

# Create and activate the virtual environment
python3 -m venv venv
source venv/bin/activate

# Run the Velos Sentry
python velos.py
```
